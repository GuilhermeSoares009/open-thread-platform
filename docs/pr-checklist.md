# PR Checklist

- [ ] PR is small, focused, and follows trunk-based development
- [ ] Spec and ADRs updated if behavior or decisions changed
- [ ] API contracts updated (OpenAPI + docs)
- [ ] Authorization policies verified
- [ ] Validation covers user input and edge cases
- [ ] Tests added or updated (unit/feature/integration)
- [ ] Lint and static analysis pass
- [ ] No secrets committed (.env.local, tokens, keys)
